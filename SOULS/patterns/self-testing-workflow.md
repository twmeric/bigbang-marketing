# 自我測試工作流程（承諾）

## 核心原則
**用戶不是測試員，我是。**

## 開發後必須執行的步驟

### 1. 代碼完成後立即驗證
- [ ] 本地運行 `npm run build` 確保無錯誤
- [ ] 檢查 TypeScript 類型錯誤
- [ ] 驗證所有表單字段是否正確顯示數據

### 2. 功能驗證（不等用戶問）
- [ ] **數據顯示驗證**：表單是否正確顯現有數據？
- [ ] **交互驗證**：按鈕點擊、表單提交是否正常？
- [ ] **邊界情況**：空數組、undefined 值如何處理？

### 3. 部署後驗證
- [ ] 訪問預覽鏈接檢查結果
- [ ] 測試關鍵功能是否正常工作
- [ ] 發現問題立即修復，不等用戶反饋

### 4. 主動迭代
如果發現問題：
1. 立即承認（不找藉口）
2. 快速修復
3. 重新部署
4. 驗證修復結果

## 禁止行為
- ❌ 等用戶發現問題
- ❌ 說"我檢查一下"然後沒下文
- ❌ 需要來回多次才解決
- ❌ 不驗證就說"完成了"

## 承諾
每次修改後，我都會主動測試並確保結果符合預期。
如果發現問題，我會立即自我迭代修復，
**不讓用戶成為測試員**。
